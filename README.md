# ドレミバー v015

鼻歌・歌声・単音メロディの音程を、カラオケの音程バーのように表示する簡易Webアプリです。

v010では、音声ファイルの読み込み・ブラウザ録音・解析モード・音名表示切替・画像保存・ドレミ音再生に加えて、録音音源をそのまま保存し、作曲アプリ向けにWAVでも書き出せるようにしました。

## 使い方

### ファイルから使う場合

1. `index.html` をブラウザで開く
2. 音声ファイルを選択
3. 「解析する」を押す
4. 音程バーが表示される

通常は「解析モード：おまかせ（おすすめ）」＋「音名表示：やさしい（ドレミだけ）」のままで大丈夫です。

### 録音して使う場合

1. 「録音開始」を押す
2. マイク使用を許可する
3. 鼻歌・歌声を録音する
4. 「録音停止」を押す
5. 録音音源を再生確認する
6. 「解析する」を押す

## 録音機能について

録音機能はブラウザのマイク機能を使います。

環境によっては、`index.html` を直接開いた場合に録音できないことがあります。
その場合は、以下のどちらかで試してください。

### 方法1：Vercelなどに公開する

HTTPSの公開URL上ではマイク機能が動きやすいです。

### 方法2：ローカルサーバーで開く

このフォルダで以下を実行します。

```bash
python -m http.server 8000
```

その後、ブラウザで以下を開きます。

```text
http://localhost:8000
```

## 表示について

固定ド表示です。

- C = ド
- D = レ
- E = ミ
- F = ファ
- G = ソ
- A = ラ
- B = シ

## v006 の変更点：ドレミ音で再生

2. 「解析する」を押す
3. 音程バーが表示される

## v009 の変更点

### 音程バー表示の方針変更

v008ではスマホ向けに音程バーを少し圧縮していましたが、v009では基本を横スクロール表示に戻しました。

- 基本：横スクロール（おすすめ）
- 補助：全体表示

全体表示は、全体の流れをざっくり見たい時に使います。
細かく音程を見る場合は、横スクロール表示がおすすめです。

### ドレミ音量の改善

ドレミ音が小さく聞こえる問題に対して、以下を変更しました。

- 音量設定を追加
  - 標準
  - 大きめ
  - 最大（スマホ向け）
- 初期値を「最大（スマホ向け）」に設定
- 正弦波だけではなく、三角波＋小さな倍音を追加
- スマホスピーカーでも聞こえやすい明るい音色に変更
- 音割れを抑えるためにコンプレッサーを追加

音が大きすぎる場合は、画面上の「ドレミ音量」を下げてください。

## v010 の変更点：録音音源の書き出し

録音した音声データを保存できるようにしました。

### 追加ボタン

- 元の音源を保存
- WAVで保存

### 元の音源を保存

ブラウザが録音した形式のまま保存します。

例：

- Chrome / Android：webm になることが多い
- Safari / iPhone：mp4 / m4a 系になることがある

端末やブラウザによって形式は変わります。

### WAVで保存

作曲アプリやDAWに入れやすいよう、読み込んだ音源をWAVとして保存します。

用途：

- 作曲アプリに読み込む
- DAWに読み込む
- 後から編集する
- 他のAI作曲ツールに渡す

### 注意

WAV保存は、録音した音声を音程補正する機能ではありません。
録音した声そのものを、WAV形式で保存する機能です。

音程バーやドレミ音再生の結果を音声化して保存する機能ではなく、作曲素材として使うための「元の録音音源」の保存です。


## v012 修正点

v011では、JavaScript側に「ドレミ音をWAV保存」の処理が入っている一方で、HTML側に対応するボタンIDが入っておらず、画面読み込み時にJavaScriptが途中で止まる可能性がありました。

v012では以下を修正しています。

- `downloadSynthWavButton` をHTMLに追加
- `synthExportStatus` をHTMLに追加
- `synthTone` の音色選択をHTMLに追加
- ドレミ音WAV保存のクリック処理をnull-safe化
- `OfflineAudioContext` が不安定な環境向けに、手動WAV生成のフォールバックを追加


## v013 修正点

保存ボタンが薄い表示のまま機能しない問題への対策として、画像保存以外の保存ボタンを「常に押せる」形式に変更しました。

### 変更内容

- 元の音源を保存：常に押せる
- 録音音源をWAV保存：常に押せる
- ドレミ音をWAV保存：常に押せる
- データがない状態で押した場合は、無効化ではなく画面メッセージで理由を表示
- 保存処理中のみ、ボタンに処理中の見た目を付与
- 保存ボタンのdisabled制御を撤廃し、状態管理をシンプル化

これにより、保存ボタンが薄いまま押せない状態になることを避けています。


## v014 修正点：スマホUI崩れ対策

解析完了後にスマホ画面が崩れる問題への対策を入れました。

### 主な修正

- 画面全体が横に広がらないように `overflow-x` を制御
- 音程バーの横スクロールを、音程バー枠内だけに閉じ込める
- スマホでは「音の流れ」の表示数を少し減らす
- 「音の流れ」は高さ制限つきスクロール表示に変更
- 保存エリア・音色設定・ボタン類が画面幅を押し広げないように調整
- 解析ミスの外れ値で音程バーが極端に縦長にならないよう、表示範囲だけを調整

解析データ自体は保持し、スマホで見える範囲のUI崩れを防ぐための調整です。


## v015 修正点：解析後も初期UI幅を保つ

v014でも解析後にスマホの横幅が変わるケースがあったため、v015では音程バー部分をより強く独立させました。

### 主な修正

- 解析後も `.app` の幅を `100vw` 基準で固定
- `result-panel` を `contain` で隔離
- `canvas-wrap` に明示的な高さと幅を設定
- 大きなcanvas幅が親要素・ページ全体の横幅計算に影響しないように調整
- 音程バーの横スクロールは、音程バー枠の中だけで発生
- `clearCanvas()` 時も安全な幅に戻すよう修正

これにより、録音・解析後も最初に表示されていたUI幅を保つことを狙っています。
