# ドレミバー v008

鼻歌・歌声・単音メロディの音程を、カラオケの音程バーのように表示する簡易Webアプリです。

v008では、音声ファイルの読み込み・ブラウザ録音・解析モード・音名表示切替・画像保存・ドレミ音再生に加えて、スマホでの見やすさとドレミ音の音量を改善しました。

## 使い方

### ファイルから使う場合

1. `index.html` をブラウザで開く
2. 音声ファイルを選択
3. 「解析する」を押す
4. 音程バーが表示される

通常は「解析モード：おまかせ（おすすめ）」＋「音名表示：やさしい（ドレミだけ）」のままで大丈夫です。

### 録音して使う場合

1. 「録音開始」を押す
2. マイク使用を許可する
3. 鼻歌・歌声を録音する
4. 「録音停止」を押す
5. 録音音源を再生確認する
6. 「解析する」を押す

## 録音機能について

録音機能はブラウザのマイク機能を使います。

環境によっては、`index.html` を直接開いた場合に録音できないことがあります。
その場合は、以下のどちらかで試してください。

### 方法1：Vercelなどに公開する

HTTPSの公開URL上ではマイク機能が動きやすいです。

### 方法2：ローカルサーバーで開く

このフォルダで以下を実行します。

```bash
python -m http.server 8000
```

その後、ブラウザで以下を開きます。

```text
http://localhost:8000
```

## 対象

- 鼻歌
- 歌声
- 単音メロディ

## 苦手なもの

- 伴奏入り音源
- 複数の音が同時に鳴る音源
- 雑音が大きい音源
- 声がかなり小さい音源

## 表示について

固定ド表示です。

- C = ド
- D = レ
- E = ミ
- F = ファ
- G = ソ
- A = ラ
- B = シ

## v006 の変更点：ドレミ音で再生

解析した音程バーを、ブラウザ上のシンプルなシンセ音で再生できます。

- 元の声ではなく、検出した音程バーをもとにしたドレミ音です
- MIDIファイルを書き出す機能ではありません
- 自分の鼻歌がどんな音の並びとして認識されたかを耳で確認できます

使い方：

1. 音源を読み込む、または録音する
2. 「解析する」を押す
3. 音程バーが表示される
4. 「ドレミ音で再生」を押す

## v007 の変更点：Web公開準備

このフォルダは静的ファイルだけで動きます。

- `index.html`
- `styles.css`
- `app.js`

そのため、GitHubにアップロードしてVercelで公開しやすい構成です。

### Vercel公開の目安

1. GitHubに新しいリポジトリを作る
2. このフォルダ内のファイルをアップロードする
3. Vercelで「Add New Project」
4. GitHubリポジトリを選ぶ
5. Framework Preset は `Other` または自動判定
6. Build Command は空欄でOK
7. Output Directory も空欄でOK
8. Deploy

## v008 の変更点

### スマホでの見やすさ改善

- スマホ表示時は音程バーの横幅を少しコンパクトに調整
- スマホ表示時は行間を少し広げて、バーとドレミの高さを見やすく調整
- 横スクロールしても音名がわかりやすいように、音名をバー内の途中にも薄く表示
- 音程バーの下に「音の流れ」を追加
  - ド / レ / ミ の並びをチップで確認できます
  - チップを押すと、その位置から元音源を再生できます

### ドレミ音の音量改善

- ドレミ音で再生する時の音量を上げました
- 音割れしにくい範囲で、以前より聞こえやすい設定にしています

## スマホ確認のポイント

- 解析後、横スクロールしても音名が見えるか
- 「音の流れ」でドレミの並びが確認しやすいか
- チップを押して該当箇所から再生できるか
- ドレミ音で再生した時、以前より音量が十分か
