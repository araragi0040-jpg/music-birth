# ドレミバー v019

## 不具合原因

v018には、Viteでビルドする前のファイルと、ビルド後の公開用ファイルが同梱されていました。

公開環境でビルド前の `index.html` がそのまま配信されると、ブラウザが次の記述を直接読み込みます。

```js
import { BasicPitch } from "@spotify/basic-pitch";
```

ブラウザはこのパッケージ名を直接解決できないため、JavaScript全体が初期化前に停止していました。

その結果、次の機能がまとめて動かない状態になっていました。

- 音声ファイル選択後の読込
- 録音開始・録音停止
- 解析
- JavaScriptを使う保存機能

## v019の修正

公開用ルートには、Viteでビルド済みの静的ファイルだけを配置しています。

```text
doremi_bar_v019/
├─ index.html
├─ assets/
│  ├─ index-....js
│  └─ index-....css
├─ model/
│  ├─ model.json
│  └─ group1-shard1of1.bin
├─ vercel.json
├─ README.md
└─ source/
   ├─ index.html
   ├─ app.js
   ├─ styles.css
   ├─ package.json
   └─ public/model/
```

`source`フォルダは今後の編集用です。
公開されるアプリは、ルートにあるビルド済みファイルを使用します。

## Vercel設定

今回はVercel側でビルドを行いません。

- Framework Preset：Other
- Build Command：空欄
- Output Directory：空欄
- Install Command：空欄

既存プロジェクトでVite設定が残っている場合は、Framework PresetをOtherへ変更してください。

## 動作確認

1. 音声ファイル選択欄を押す
2. MP3・WAV・M4Aなどを選ぶ
3. 音声プレイヤーへ反映されることを確認
4. 録音開始を押す
5. マイク許可を承認
6. 録音停止後に音声プレイヤーへ反映されることを確認
7. Basic Pitchまたは通常解析を実行

録音はHTTPSまたはlocalhostで利用してください。

## v022 Basic Pitch解析修正

以下の4点のみ修正しています。

1. Basic Pitchの最大周波数1000Hz制限を解除
2. 前の音を優先する連続性補正を廃止
3. energyToleranceを下げ、前の音の残りを短縮
4. Basic Pitch解析の最短セグメントを0.05秒へ統一

UI・保存・録音・編集・通常解析など、その他の内容は変更していません。


## v023 高音表示修正

Basic Pitchが検出した最高音を、表示範囲から除外しないよう修正しました。

従来は、検出音域が広い場合に上下の一部を表示範囲から外し、
さらに中央を基準に約2オクターブ半へ縮めていました。
そのため、検出済みの上の「シ・ド」が描画範囲外になり、
音程バーに表示されない場合がありました。

v023では次の処理に変更しています。

- 検出済みの最高音は必ず表示する
- 音域が広すぎる場合は低音側の外れ値だけを調整する
- 最高音を基準に表示範囲を確保する

Basic Pitch解析設定、録音、保存、バー編集、UIなどは変更していません。


## v024 ドレミ音再生の改善

解析後のバーが細かく大量に生成された場合、従来は各バーごとに複数の
Oscillator / Gain / Filterノードを一括生成していました。

Basic Pitchの解析結果によってはWeb Audioノード数が非常に多くなり、
スマートフォンや一部ブラウザで、再生位置だけ進んで音が出ない場合がありました。

v024では次の点だけを改善しています。

- 解析用AudioContextとドレミ音再生用AudioContextを分離
- ドレミ音を事前に1本のAudioBufferへ合成
- 1つのAudioBufferSourceNodeで再生
- AudioContextがrunning状態になったことを確認してから再生
- モバイル向けのAudioContext解除処理を追加

録音音源、解析、表示、保存、バー編集、UIは変更していません。
